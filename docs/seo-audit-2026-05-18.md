# SEO / GSC Indexing Audit — 2026-05-18

Source: `python3 scripts/gsc-indexing-audit.py` against `sc-domain:esdeveniments.cat`.

Coverage signals: Search Analytics + URL Inspection + Sitemap submissions APIs.

## TL;DR

1. **Staging subdomain is openly crawlable by Google** — `staging.esdeveniments.cat/*` URLs are in GSC with status "Crawled - currently not indexed", robots ALLOWED. Wastes crawl budget, risks duplicate-content competition with `www.`. Block via `robots.txt` / `X-Robots-Tag` keyed off env, or HTTP basic auth.
2. **GSC Sitemaps API `indexed` field is unreliable** — reports `0` for all 4 sitemaps even though URL Inspection confirms production URLs are "Submitted and indexed". Don't read indexed-count from `/sitemaps`; use URL Inspection or the search-analytics indexed-URL query instead.
3. **8,359 warnings on `/server-sitemap.xml`** — needs investigation (the other three sitemaps are clean).
4. **The `place` bucket validates the PR #313 thesis** — bare `/[place]` pages: +61 pages crawled, **−28.4% clicks** in 28 days. The "thin payload → low CTR → de-prioritized" pattern PR #313 targets. This row is the baseline for measuring impact 4 weeks post-deploy.
5. **EN locale is bleeding** — 5 of top 15 impression-drop URLs are `/en/...`. Separate problem; not addressed here.
6. **Homepage clicks −69.2%** and **news-detail impressions −75.8%** — separate, unaddressed issues. Worth their own investigations.

## Action items

1. **Block staging subdomain from crawlers** (high, easy). Options, in order of preference:
   - `X-Robots-Tag: noindex, nofollow` header in `proxy.ts` when `NEXT_PUBLIC_VERCEL_ENV !== "production"` (also covers PR preview deployments).
   - `Disallow: /` in staging's `robots.txt`.
   - HTTP basic auth at the edge (Coolify / Cloudflare).
2. **Investigate 8,359 `/server-sitemap.xml` warnings**. Likely soft-404 or canonical-redirect URLs in the sitemap. Inspect a sample via GSC URL Inspection.
3. **Measure `place` bucket weekly post-deploy of PR #313**. Expected direction: clicks recover, pages-crawled may shrink as thin-place filter URLs de-index. Re-run `gsc-indexing-audit.py` at +2w, +4w, +8w.
4. **Audit EN locale separately**. Possible causes: content thinness, hreflang issues, or Google deprioritizing EN for CAT/ES inventory. Not addressed by PR #313.
5. **Decide fate of `/sitemap/...` archive pages**. They're own-site archive listings (not XML), 984 pages indexed, losing relevance (−29.9% impressions). Keep, noindex, or improve.
6. **Investigate homepage −69.2% clicks** — separate issue, not addressed.
7. **Investigate news-detail −75.8% impressions** — separate issue, not addressed.
